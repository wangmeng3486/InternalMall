# 企业内部福利商城 - 后端架构信息

|- **更新日期**: 2026-02-26
|- **技术栈**: Node.js + Express.js + SQLite + JWT + bcryptjs
|- **状态**: 开发中

## 目录结构

```
backend/
├── src/
│   ├── controllers/
│   │   ├── adminController.js    # 管理员控制器(登录/系统状态/商品CRUD/分类CRUD/图片上传/库存预警/热销)
│   │   ├── cartController.js     # 购物车控制器(列表/添加/更新/删除/批量/统计)
│   │   └── productController.js  # 商品控制器(列表/详情/搜索/分类/收藏)
│   ├── services/
│   │   ├── cartService.js        # 购物车业务逻辑
│   │   └── productService.js     # 商品业务逻辑(含分类/收藏/统计)
│   ├── models/
│   │   ├── Cart.js               # 购物车数据模型(操作cart表)
│   │   ├── Category.js           # 分类数据模型(操作categories表)
│   │   └── Product.js            # 商品数据模型(操作products表)
│   ├── routes/
│   │   ├── auth.js               # 认证路由(登录/登出逻辑直接写在路由中，无独立controller)
│   │   ├── admin.js              # 管理员路由 → adminController
│   │   ├── cart.js               # 购物车路由 → cartController
│   │   ├── health.js             # 健康检查路由(逻辑直接写在路由中)
│   │   ├── product.js            # 商品路由 → productController
│   │   └── user.js               # 用户路由(逻辑直接写在路由中，无独立controller)
│   ├── middleware/
│   │   ├── auth.js               # 认证中间件(authenticateToken/requireAdmin/requireUser)
│   │   └── error.js              # 全局错误处理中间件
│   ├── config/
│   │   ├── app.js                # 应用配置(端口/JWT/CORS/分页等)
│   │   └── database.js           # SQLite连接管理(db/query/get/run/close)
│   ├── database/
│   │   └── init.js               # 数据库初始化(建表+种子数据)
│   └── utils/
│       ├── fileUpload.js         # 文件上传(multer+sharp，单图/多图中间件)
│       └── response.js           # 统一响应格式(successResponse/errorResponse)
├── database/
│   └── mall.db                   # SQLite数据库文件
├── test/
│   ├── cart-api.js               # 购物车API集成测试(axios，手动运行)
│   └── product-api.js            # 商品API集成测试(mocha风格，需chai)
├── uploads/
│   └── products/                 # 商品图片存储目录
├── .env                          # 环境变量
├── init-db.js                    # 数据库初始化入口脚本
├── mall.db                       # 根目录遗留数据库文件(0字节，实际数据库在database/mall.db)
├── test-api.js                   # 简易API测试脚本(原生http模块)
├── package.json
└── server.js                     # 服务器入口(路由注册/中间件/静态文件服务)
```

## 分层说明

当前分层不统一：
- **商品模块、购物车模块**：走完整的 Route → Controller → Service → Model 四层
- **认证模块、用户模块、健康检查**：业务逻辑直接写在 routes/ 文件中，无 Controller/Service 层

## 路由注册 (server.js)

```
/api/health    → routes/health.js
/api/auth      → routes/auth.js
/api/user      → routes/user.js
/api/admin     → routes/admin.js
/api/products  → routes/product.js
/api/cart      → routes/cart.js
```

## 模块依赖关系

- **routes/admin.js** → adminController → productService → Product.js + Category.js
- **routes/cart.js** → cartController → cartService → Cart.js
- **routes/product.js** → productController → productService → Product.js + Category.js
- **routes/auth.js** → 直接使用 database.js + bcryptjs + jsonwebtoken
- **routes/user.js** → 直接使用 database.js
- **middleware/auth.js** → 被 admin/cart/product/user 路由使用
